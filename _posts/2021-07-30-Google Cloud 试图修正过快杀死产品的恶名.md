[Google Cloud 试图修正过快杀死产品的恶名](https://www.solidot.org/story?sid=68413)
------

<div>
Google 杀死的产品是如此之多以至于它每推出一款新产品其他人都在猜测该产品会在什么时候死亡。这一名声影响到了用户对 Google 产品的使用率，因为被 Google 杀死的每一个产品使用者都投入了时间精力甚至金钱，如果产品免不了死亡，为什么用户要浪费时间在上面？Google 云服务 Cloud Platform 也有类似的名声，原因是它过快的淘汰 API。为了<a href="https://arstechnica.com/gadgets/2021/07/google-cloud-offers-a-model-for-fixing-googles-product-killing-reputation/" target="_blank">解决这个问题</a>，Google Cloud 宣布了 <a href="https://cloud.google.com/blog/topics/inside-google-cloud/new-api-stability-tenets-govern-google-enterprise-apis">Enterprise API</a>，承诺对特定 API 保持其稳定性。Google 称只要用户活跃的使用某个 API 功能，Google 将尽最大可能不会移除或改变功能；如果一个破坏性变化不可避免，Google 将努力确保迁移非常轻松。如果 Google 要改变某个 API，客户将至少提前一年得到通知。Google 的云服务落在竞争对手亚马逊和微软后面，其市场占有率只有 7%，而微软 19%，亚马逊 32%。此前有流言称，如果 Cloud Platform 在 2023 年前没有超过微软和亚马逊，它可能会失去资金。<p><img src="https://img.solidot.org//0/446/liiLIZF8Uh6yM.jpg" height="120" style="display:block"/></p>
</div>
